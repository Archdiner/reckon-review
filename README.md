# Reckon Review

**explain what you merge**

</div>

---

A GitHub App that **blocks a pull request from merging until a human can explain, in their
own words, the mechanism of what it changes.** Any PR, whoever (or whatever) wrote it.

```
   AI reviewers                          Reckon Review
   в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ                         в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
   "is the CODE good?"                   "does a HUMAN get it?"
   bugs, style, security                 comprehension
                         в•І             в•ұ
                          в•І           в•ұ
                     they STACK вҖ” one checks the code,
                     the other checks the person shipping it
```

> Free during the beta, use it!

## Install it В· ~30 seconds

```
  1  install    в–ё github.com/apps/reckon-review/installations/new
                  (or: repo вҶ’ Settings вҶ’ GitHub Apps вҶ’ Configure)

  2  done       в–ё that's it. On install Reckon creates a branch ruleset on your
                  default branch that REQUIRES the "Reckon comprehension" check вҖ”
                  so merges are BLOCKED until someone explains the PR. By default.
                  No branch-protection step to remember.
```

>  **Gating is on by default.** Reckon needs the `Administration` permission (granted at
> install) to set the required-check ruleset for you. Because it uses GitHub *rulesets*, the
> check gates the very first PR вҖ” no "run once to register it" dance. If Reckon can't set the
> ruleset (permission not granted), it falls back to advisory: you'll see the вң— but the merge
> button still works.

> **Want advisory-only instead?** Delete the ruleset Reckon created
> (Settings вҶ’ Rules вҶ’ Rulesets вҶ’ **Reckon comprehension (managed)** вҶ’ delete). Reckon keeps
> posting its check and comments; it just won't block the merge.

## What it can (and can't) see

```
   reads   в–ё the PR's diff + the explanation comments people write
   writes  в–ё a status check + comments on the PR
   never   в–ё reads code outside the PR В· stores your source В· touches anything else
```

## FAQ

```
  Cost?              free during the beta - we cover the grading.
  Not blocking?      Reckon couldn't set the ruleset - it needs the "Administration"
                     permission (granted at install) to require its own check. Re-check
                     the App's permissions, or add the required check manually.
  Advisory only?     delete the "Reckon comprehension (managed)" ruleset. the check still
                     posts, it just stops blocking the merge.
  Who explains it?   any one reviewer with write access. one good explanation clears it.
  Interfere with CI? no. a separate status check alongside your CI. it runs no code.
  Turn it off?       uninstall the App, or delete the managed ruleset.
```

**Beta** вҖ” if the grader ever seems wrong or the flow feels off, that's exactly the feedback
we want. Open an issue.
