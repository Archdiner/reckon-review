# Reckon Review

**explain what you merge**

</div>

---

A GitHub App that **blocks a pull request from merging until a human can explain, in their
own words, the mechanism of what it changes.** Any PR, whoever (or whatever) wrote it.

```
   AI reviewers                          Reckon Review
   в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ                         в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
   "is the CODE good?"                   "does a HUMAN get it?"
   bugs, style, security                 comprehension
                         в•І             в•ұ
                          в•І           в•ұ
                     they STACK вҖ” one checks the code,
                     the other checks the person shipping it
```

> рҹҰҺ **Hosted for you.** Nothing to run, no server, no keys to manage. Free during the beta.

## Install it В· ~1 minute

```
  1  install    в–ё github.com/apps/reckon-review/installations/new
                  (or: repo вҶ’ Settings вҶ’ GitHub Apps вҶ’ Configure)

  2  open a PR   в–ё Reckon Review posts its check + elicit comment.
                  this first run REGISTERS the "Reckon comprehension" check
                  so GitHub will let you require it in the next step.

  3  require it  в–ё this is the step that makes it actually BLOCK a merge.
                  Rulesets (recommended):
                    Settings вҶ’ Rules вҶ’ Rulesets вҶ’ New branch ruleset
                    вҶ’ target: default branch вҶ’ Require status checks to pass
                    вҶ’ add  "Reckon comprehension"  вҶ’ Enforcement: Active
                  or classic:
                    Settings вҶ’ Branches вҶ’ Add rule вҶ’ pattern: main
                    вҶ’ Require status checks to pass вҶ’ select "Reckon comprehension"
```

> вҡ пёҸ **Until step 3, Reckon is advisory.** A GitHub App can only *report* a status
> check вҖ” GitHub only *enforces* it (blocks the merge) once branch protection requires
> that check. Skip step 3 and you'll see a red вң— but the merge button still works.
> The check only appears in the list *after* it has run once, which is why step 2 comes
> first.

> рҹ”ң **Coming soon:** opt-in auto-configuration so Reckon sets the required check for you
> on install вҖ” no manual branch-protection step.

## What it can (and can't) see

```
   reads   в–ё the PR's diff + the explanation comments people write
   writes  в–ё a status check + comments on the PR
   never   в–ё reads code outside the PR В· stores your source В· touches anything else
```

## FAQ

```
  Cost?              free during the beta - we cover the grading.
  Not blocking?      you skipped step 3 - require the "Reckon comprehension" check in
                     branch protection. without that, GitHub leaves it advisory.
  Who explains it?   any one reviewer with write access. one good explanation clears it.
  Interfere with CI? no. a separate status check alongside your CI. it runs no code.
  Turn it off?       uninstall the App, or drop the required check.
```

**Beta** вҖ” if the grader ever seems wrong or the flow feels off, that's exactly the feedback
we want. Open an issue.
